## Composure Conventions

Both agents coordinate through shared files. Follow these conventions to avoid stepping on each other's work.

### Agent Activity Log

At the **start** of every work session, append to `sessions/agent_activity.md`:
```markdown
### [ISO datetime] — AG — Session Start
- Workspace: TommyTalker
- Model: [current model]
- Plan: [brief description of what you'll work on]
```

At the **end** of every session (or when TJ runs `/wrapup`), append:
```markdown
### [ISO datetime] — AG — Session End
- Files changed: [list]
- Handoff notes: [anything CCT should know]
```

### Before Editing Shared Files

1. Check `sessions/agent_activity.md` — is CCT currently active?
2. If CCT is active and working in the same area, warn TJ before proceeding
3. Log your edits in the activity log

### Domain Lanes

| Domain | Your Role | Notes |
|--------|-----------|-------|
| `src/`, `tests/` | As TJ assigns | Either agent |
| `_project/` | Read and write | Planning docs |
| `sessions/` | Append-only | Both agents |
| `assets/` | Primary | Image generation, visual assets |
| Git operations | Read status only | CCT commits with security scan |

### Git Convention

**Do not commit changes.** Leave them uncommitted for CCT to review and commit during its next session. CCT runs a security scan before committing.

**Exception:** If TJ explicitly tells you to commit in a solo AG session, note the commit in `sessions/agent_activity.md`.

### Scratchpad

`sessions/scratchpad.md` is a shared working area for inter-agent notes. Use it to:
- Leave URLs, research findings, or code snippets for CCT
- Note tasks that need CCT's attention
- Read notes CCT left for you

### Rate Limit Protocol

If you hit rate limits:
1. Log in `sessions/agent_activity.md` with timestamp
2. If mid-workflow, write the pending task to `sessions/scratchpad.md` so CCT or the next session can continue
3. Note model switches in `sessions/agent_activity.md`
4. Do not block CCT's work — continue with reduced capability or hand off

### Mid-Session Check

`/gogogo` only runs at session start. If CCT writes to `sessions/scratchpad.md` while you're already working, you won't see it until the next session.

**Convention:** Before starting a new deliverable or major task within a session, re-read `sessions/scratchpad.md` to check for notes that arrived since your last check.
